附录 A — 站点说明与补充资源

这一页不再作为外部资源导流页，而是作为 R语言实战医学统计 站点自己的补充说明页，集中回答“本站是什么、怎么用、适合谁、后续怎么验证和维护”这些问题。

A.1 站点定位

本站是一个面向医学科研与临床研究场景的 R 语言统计学习站。它的核心目标不是替代教材，也不是替代统计软件，而是把常见医学统计方法整理成：

  • 可直接阅读的中文教程；
  • 可复制的 R 代码；
  • 配套的交互式统计图与计算器；
  • 适合复现教材例题与论文方法部分的学习入口。

如果你的目标是复习医学统计、复现教材例题、写论文方法部分，或者快速找到某种常见统计分析在 R 中的实现方式，这个站点就是为这些任务设计的。

A.2 推荐使用方式

  1. 先从首页按“研究问题”或“学习路线”进入对应章节；
  2. 先阅读统计方法的适用条件，再看代码实现；
  3. 把代码复制到本地 R / RStudio 中运行；
  4. 结合页面中的交互式组件理解参数变化如何影响结果；
  5. 真正用于论文或项目时，再回到原始数据与研究设计核对前提条件。
提示

本站更适合做“方法理解 + 代码复现 + 输出解释”的桥梁，不建议在没有核对研究设计与假设前提的情况下直接把某段代码机械套用到正式研究中。

A.3 章节结构速览

  • 基础统计分析：适合从 t 检验、方差分析、卡方检验、秩和检验、相关回归开始打基础；
  • 高级统计分析：适合处理 Logistic 回归、生存分析、聚类、主成分、结构方程等更复杂问题；
  • 文献常见统计分析：适合论文复现与较新的临床研究分析场景，例如倾向评分、竞争风险、样条回归、亚组分析。

建议零基础用户先完成基础部分，再进入高级和文献部分；已经有统计基础的用户可以直接从首页“按研究问题查找”进入对应专题。

A.4 验证与维护说明

本站除了内容页面，还包含一套轻量的回归校验基线，用于保护关键统计计算器不在后续维护中被悄悄改坏。

  • tests/stat_calculator_cases.json：关键组件的样例输入与预期约束；
  • tests/run_validation.js：零依赖最小可执行验证脚本；
  • npm run validate：统一验证入口。

目前优先覆盖的组件包括：

  • ttest
  • chisq / Fisher exact
  • kruskal
  • friedman
  • Kaplan-Meier 生存曲线的基本不变量

这套验证不是完整自动化测试框架，但已经能在重构后帮助快速发现一些高风险统计逻辑回归。

A.5 本地开发入口

当前项目已经提供最小统一命令入口：

npm run preview
npm run validate

其中：

  • npm run preview：本地启动一个静态文件服务器；
  • npm run validate：运行当前最小可执行验证脚本。
